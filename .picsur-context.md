# Picsur Project Context

## Overview

Picsur is an open-source image hosting service (like Imgur) built with a modern TypeScript monorepo architecture. It is **NOT actively maintained** according to the README, but remains functional.

## Tech Stack

- **Frontend**: Angular 18, Bootstrap, SCSS
- **Backend**: NestJS with Fastify, TypeORM
- **Database**: PostgreSQL (via Docker)
- **Package Manager**: pnpm 9.6.0
- **Node Version**: v20 (see .nvmrc)
- **Build System**: Multi-stage Docker builds for ARM64 and AMD64

## Key Dependencies

- **Image Processing**: sharp (0.33.5), qoi-img (2.2.0), bmp-img (1.3.0)
- **Authentication**: passport-jwt, bcrypt-ts
- **HTTP**: axios (1.15.0), fastify (4.28.1)
- **Database**: TypeORM, pg
- **Validation**: Zod (3.23.8)

## Build Commands

```bash
pnpm install  # Install all dependencies
pnpm -r build # Build all packages (shared, backend, frontend)
pnpm lint     # Run ESLint on all packages
pnpm format   # Format code with Prettier
```

## Docker Commands

```bash
pnpm devdb:start   # Start PostgreSQL dev database
pnpm devdb:stop    # Stop PostgreSQL dev database
pnpm devdb:restart # Restart database
```

## Security Notes

- **Axios**: Updated from 1.7.7 to 1.15.0 to fix CVE-2025-62718 and CVE-2025-58754
- **Moment.js**: Uses 2.30.1 (patched for CVE-2022-31129)
- Project has no unit tests (no spec/test files found)
- ESLint 9.x required migration from .eslintrc.cjs to eslint.config.js

## Known Issues Fixed

1. **Webpack config**: Changed from ESM to CommonJS (.cjs) for @angular-builders/custom-webpack
2. **ESLint config**: Migrated to flat config format for ESLint 9.x
